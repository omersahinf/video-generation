# Current State — 2026-05-27

## Status: REVEAL-MASK ANIMATION ENGINE ✅

### Core Animation: Contour-Ordered Reveal Mask
- cv2.findContours extracts real stroke paths from illustration
- Nearest-neighbor contour ordering keeps the drawing path spatially natural
- Contours are drawn into masks only; frames reveal original illustration pixels
- Phase 1: narrow line mask reveals original stroke pixels
- Phase 2: dilated fill mask reveals nearby color pixels
- Final frame uses the same mask blend path, so there is no hard switch to a clean image
- User confirmed: "mantık çok iyi, bu mantıkla ilerlememiz gerekiyor"

### Mandatory Output Rules
- Default video language is English for all future video generation.
- The prompt language does not control output language. Turkish prompts still produce English scripts, English on-screen text, and English voiceover unless the user explicitly asks for another output language.
- Normal video requests must use the full production pipeline: Ollama script → AI image generation with configured API token → Edge TTS → contour reveal animation → MoviePy composition.
- Do not deliver standalone synthetic test videos or low-quality fallback-only renders for normal video requests.
- Do not add detached title bars or post-generation text overlays. On-screen words must be generated by the image model as natural hand-drawn labels, signs, callouts, speech bubbles, arrows, or object labels inside the scene.
- Fill animation no longer jumps to the full foreground at the end. Ink is contour-driven; color/light gray regions reveal as full-opacity scheduled components near their contours.
